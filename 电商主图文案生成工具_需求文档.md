# 电商主图 & 文案自动生成工具
## 业务逻辑文档 + AI IDE 原型提示词

---

## 一、产品背景

电商运营在日常工作中需要为每一款商品手动制作主图、撰写标题和卖点文案，流程繁琐、重复劳动多，且对创意输出和执行效率都有较高要求。本工具的目标是：**让运营上传商品基本信息后，系统自动生成一批可用草稿，运营只需挑选、微调即可直接使用**，将"从零开始"变成"二次加工"，大幅提升效率。

---

## 二、目标用户

| 角色 | 使用场景 |
|------|----------|
| 电商运营 | 新品上架、活动换图、日常更新商品详情 |
| 店铺主理人 | 小团队自运营，没有专职设计/文案 |
| 品牌内容团队 | 批量生产 SKU 素材时的初稿工具 |

---

## 三、核心功能（MVP 版本）

### 3.1 输入侧（左栏）

| 字段 | 类型 | 说明 |
|------|------|------|
| 商品名称 | 文本输入 | 例：「冰丝透气防晒外套」 |
| 商品类目 | 下拉选择 | 服装、数码、食品、美妆、家居等 |
| 核心卖点 | 多行文本 | 运营填写 3-5 条关键词或短句 |
| 目标人群 | 单选/文本 | 例：18-25 岁女生、户外运动爱好者 |
| 价格区间 | 文本输入 | 例：「79 元」或「99-199 元」 |
| 促销信息 | 文本输入（可选）| 例：「限时 5 折」「买一送一」 |
| 商品主图 | 图片上传 | 支持 JPG/PNG，作为生成主图的参考底图 |
| 风格偏好 | 单选标签 | 简约/国潮/高端/活泼/促销爆款 |

### 3.2 输出侧（右栏）

系统生成后，右侧展示以下内容：

#### 主图预览区
- 展示 **2-3 张** 不同布局风格的主图构图方案（卡片形式）
- 每张包含：商品图 + 标题文字 + 卖点标签的视觉合成预览
- 支持点击放大预览

#### 文案区
- **商品标题**：2-3 个版本（搜索优化型 / 情感共鸣型 / 促销冲动型）
- **卖点文案**：3-5 条短句，适配主图角标或详情页 banner
- **种草长文**：100-200 字的小红书/朋友圈风格种草段落（可选展示）

#### 操作按钮
- 「复制文案」「重新生成」「调整风格后重新生成」
- 每个文案版本旁有「采用此版本」标记

---

## 四、业务流程

```
运营填写商品信息 + 上传主图
        ↓
点击「生成草稿」
        ↓
系统调用 AI 生成文案（标题 × 3、卖点 × 5、种草文 × 1）
系统根据商品图 + 文案生成主图构图预览 × 3
        ↓
右侧展示所有草稿结果
        ↓
运营浏览 → 选择满意版本 → 复制使用 / 继续微调
        ↓
（可选）调整参数后重新生成
```

---

## 五、数据流说明

- 前端收集表单数据 + 图片（base64）
- 调用后端 API（或直接调用 Claude API）生成文案
- 主图预览为前端 Canvas / CSS 合成，将商品图与生成文字叠加渲染
- 无需登录、无需持久化存储（MVP 阶段，刷新即清空）

---

## 六、非功能性要求（MVP）

- 响应时间：文案生成 < 10 秒，主图预览渲染 < 3 秒
- 兼容性：Chrome / Edge 最新版桌面浏览器
- 图片上传限制：单张 ≤ 5MB

---

## 七、后续迭代方向（非 MVP）

1. 批量模式：CSV 导入多个 SKU，批量生成
2. 模板库：保存常用风格模板，下次一键复用
3. 导出功能：下载合成主图 PNG / 复制 Markdown 文案
4. 历史记录：保存每次生成结果，方便回溯
5. 团队协作：多人共用，支持评论和标注

# AI IDE 原型提示词

> 以下提示词可直接粘贴到 Cursor、Trae、Windsurf 等 AI IDE 中，生成可运行的单页面应用原型。

---

```markdown
# 任务目标
为电商运营团队开发一个「商品主图 & 文案自动生成工具」的单页面应用（SPA）原型。
技术栈：React + Tailwind CSS（或纯 HTML/CSS/JS，优先 React）。
使用 Anthropic Claude API 实现 AI 文案生成能力（claude-sonnet-4-20250514 模型）。

---

## 页面整体布局

左右分栏，宽度比例约 4:6（左侧输入区 / 右侧结果区）。
页面顶部有工具名称和简短说明。
整体视觉风格：现代、干净、专业，适合 B 端工具，主色调可选深蓝或深灰，搭配明亮强调色。

---

## 左栏：商品信息输入区

包含以下表单字段：

1. **商品名称**（必填）- 单行文本输入框，placeholder: "例：冰丝透气防晒外套"
2. **商品类目**（必填）- 下拉选择，选项：服装、数码、食品、美妆、家居、运动户外、母婴、其他
3. **核心卖点**（必填）- 多行文本域，4 行高，placeholder: "每行写一条卖点，例：\n轻薄透气\n防紫外线 UPF50+\n多色可选"
4. **目标人群**（可选）- 单行文本，placeholder: "例：18-25 岁女生、上班族"
5. **价格 / 促销信息**（可选）- 单行文本，placeholder: "例：79元，限时5折"
6. **品牌风格**（必填）- 横向标签单选，选项：简约清新 / 国潮活力 / 高端质感 / 促销爆款 / 温柔治愈
7. **商品图片**（可选）- 拖拽上传区域，支持 JPG/PNG，上传后在区域内显示缩略图预览，限制 5MB 以内

底部放置一个醒目的「✨ 生成草稿」主操作按钮，宽度撑满左栏，点击后触发生成流程。
生成中时按钮显示 loading 状态并禁用，文案改为「生成中…」。

---

## 右栏：生成结果展示区

初始状态显示空态插画或提示文字：「填写左侧信息，点击生成即可获得文案草稿 →」

生成完成后，展示以下内容模块，每个模块有清晰的标题和分隔：

### 模块 1：商品标题（3 个版本）
- 每个版本占一行，标注类型标签（搜索优化型 / 情感共鸣型 / 促销冲动型）
- 每行右侧有「复制」按钮，点击后按钮变为「已复制 ✓」并 2 秒后还原

### 模块 2：主图卖点角标（5 条短句）
- 列表展示，每条 15 字以内
- 整体有「一键复制全部」按钮

### 模块 3：种草文案（1 段，100-200 字）
- 多行文本展示区，右上角有「复制」按钮
- 下方小字注明适用场景：「适合小红书笔记、朋友圈、详情页顶部」

### 模块 4：主图构图预览（2-3 个方案）
- 用 Canvas 或 CSS 合成渲染
- 每个预览卡片固定尺寸 800×800px（缩放显示，实际展示约 200×200）
- 合成逻辑：将用户上传的商品图作为背景（居中裁剪），叠加：
  - 顶部或底部色块区域，写上商品标题（从模块1取一个版本）
  - 角标气泡展示 1-2 条卖点（从模块2取）
  - 如用户未上传图，用品类相关的纯色渐变背景占位
- 每张预览卡片下方有「下载此图」按钮（使用 Canvas toDataURL 实现下载）
- 3 个方案在构图布局/色块位置/字体大小上有视觉差异

---

## 右栏底部操作区

- 「🔄 重新生成文案」按钮（不清空表单，重新调用 API）
- 「🗑 清空结果」按钮（清除右侧所有内容，恢复空态）

---

## AI 文案生成逻辑（前端直接调用 Claude API）

API 端点：https://api.anthropic.com/v1/messages
模型：claude-sonnet-4-20250514
max_tokens: 1500

构建 prompt 的方式：将表单数据拼入 user message，system prompt 设定角色为「资深电商运营文案专家」。

要求模型返回严格的 JSON 格式（在 system prompt 中明确），结构如下：
{
  "titles": [
    {"type": "搜索优化型", "text": "..."},
    {"type": "情感共鸣型", "text": "..."},
    {"type": "促销冲动型", "text": "..."}
  ],
  "bullets": ["...", "...", "...", "...", "..."],
  "story": "..."
}

收到响应后解析 JSON，分别填充到对应 UI 模块中。
如果解析失败，在右栏顶部显示错误提示 banner，内容为「生成失败，请检查网络或稍后重试」。

API Key 处理方式：页面顶部提供一个可收起的「⚙ 设置」区域，用户可以在其中粘贴自己的 Anthropic API Key，存入 React state（不做持久化，刷新即清空）。如果 Key 为空，点击生成按钮时弹出提示引导用户先填写 Key。

---

## 交互细节要求

1. 表单验证：点击生成时，若必填字段为空，对应输入框显示红色边框 + 提示文字
2. Loading 状态：生成中在右栏中央显示 spinner + 「AI 正在创作中，请稍候…」
3. 复制功能：使用 navigator.clipboard.writeText 实现，降级方案用 document.execCommand
4. 图片上传：拖拽和点击都支持，上传后立即渲染缩略图并触发主图预览更新
5. 响应式：最低支持 1280px 宽度的桌面浏览器，移动端不做适配

---

## 代码要求

- 使用 React 函数组件 + Hooks（useState, useRef, useEffect, useCallback）
- 样式使用 Tailwind CSS utility class（不写额外 CSS 文件）
- 将 Claude API 调用封装为独立的 async function：generateCopy(formData, apiKey)
- 将主图合成渲染封装为独立函数：renderPreviewCanvas(canvasEl, productImage, title, bullets, style)
- 组件拆分建议：App > InputPanel > ResultPanel > TitleCard > BulletsCard > StoryCard > ImagePreviewCard
- 每个主要功能区域加上注释说明

---

## 不需要实现的功能（MVP 范围外）

- 用户登录 / 鉴权
- 后端 API / 数据库
- 历史记录持久化
- 批量导入
```
